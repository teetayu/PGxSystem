# SWU-PGx Digital Platform

## 1. หน้าหลัก (Homepage / Dashboard)

### ส่วนใช้งานด่วน (Quick Access):

- ปุ่ม "เพิ่มเคสใหม่" / "สแกนใบสั่งตรวจ" ✔️
- ช่อง "ค้นหาผู้ป่วย" (HN, ชื่อ, Barcode) ✔️

### ส่วนแสดงผลภาพรวม (Dashboard Stats):

- สถิติการใช้งานรายวัน/สัปดาห์ ✔️
- [เพิ่มเติม] วิดเจ็ตติดตาม TAT: แสดงเคสที่กำลังดำเนินการ และเคสที่เสี่ยงเกิน SLA (Service Level Agreement)
- [เพิ่มเติม] วิดเจ็ต KPI คุณภาพ: เช่น อัตราการปฏิเสธสิ่งส่งตรวจ ✔️

### ส่วนการแจ้งเตือน (Notifications / Tasks):

- รายการเคสรอแปลผล / เคสเกินกำหนด TAT ✔️

### ส่วนคลังความรู้ (Knowledge Hub):

- แสดงบทความ/เปเปอร์งานวิจัยล่าสุด ✔️

## 2. เมนู: การจัดการเคส (Case Management)

- **รายการผู้ป่วย/เคส:** ตารางแสดงผู้ป่วยทั้งหมด พร้อมสถานะ TAT (pre-analytic, analytic, post-analytic) ✔️
- **เพิ่มผู้ป่วยใหม่:** ฟอร์มสำหรับกรอกข้อมูล
  - ปุ่ม "บันทึกและพิมพ์ใบสั่งตรวจ (PDF)" ✔️
- **หน้ารายละเอียดผู้ป่วย:**
  - ข้อมูลผู้ป่วย ✔️
  - [เพิ่มเติม] พิมพ์ฉลากบาร์โค้ด/QR สิ่งส่งตรวจ ✔️
  - ผลการตรวจพันธุกรรม (Genotype) ✔️
  - โมดูลแปลผล (CDS) และคำแนะนำ ✔️
  - [เพิ่มเติม] โมดูลติดตาม TDM: (สำหรับยา เช่น TPMT enzyme activity, Warfarin, etc. พร้อมผูกผลระดับยา)
  - สร้าง/ดูรายงานผล (PDF/HTML ภาษาไทย)
  - จัดการ e-Consent
  - Log การเข้าถึง/แก้ไขข้อมูลผู้ป่วย ✔️

## 3. [ใหม่] เมนู: รับสิ่งส่งตรวจ (Specimen Accessioning)

- **[เพิ่มเติม] แค็ตตาล็อกสิ่งส่งตรวจ:** (อ้างอิงฟอร์มรามาธิบดี) ✔️
  - ชนิดสิ่งส่งตรวจ, ปริมาณขั้นต่ำ, ภาชนะ, อุณหภูมิขนส่ง ✔️
  - เกณฑ์การรับ-ปฏิเสธ (Rejection Criteria)
  - Chain-of-custody
- **[เพิ่มเติม] หน้าสแกนรับเข้า (Accessioning):**
  - สแกนรับเข้า (เทียบใบสั่งตรวจ)
  - บันทึกการปฏิเสธสิ่งส่งตรวจ (พร้อมเหตุผล)
  - พิมพ์บาร์โค้ดสำหรับใช้ภายในแลป

---

## 4. เมนู: คลังข้อมูล (Knowledge & Info)

- **ข้อมูลบริการตรวจ (Test Information):**
  - ดาวน์โหลดใบสั่งตรวจ (PDF)
  - TAT (SLA) ของแต่ละการตรวจ
  - ข้อปฏิบัติก่อนตรวจ, ชนิดสิ่งส่งตรวจ, ปริมาณ
- **[เพิ่มเติม] กฎการแปลผล (CDS Rules):**
  - ฐานข้อมูล genotype -> predicted phenotype -> clinical guidance/drug recommendation
  - อ้างอิง CPIC/PharmGKB (พร้อมเวอร์ชันนิ่งและหลักฐานอ้างอิง)
  - [เพิ่มเติม] รายการเตือนความเสี่ยง HLA-ยา (เช่น HLA-B*15:02 + Carbamazepine)
  - [เพิ่มเติม] คำแนะนำการปรับขนาดยา (เช่น Warfarin, PPIs, Tacrolimus)
- **บทความและงานวิจัย:**
  - บทความให้ความรู้ / เปเปอร์งานวิจัย / ข้อมูลทุนวิจัย

## 5. เมนู: ควบคุมคุณภาพและอบรม (QC & Training)

- **[เพิ่มเติม] คลัง SOP:**
  - ระบบจัดการเวอร์ชัน, อนุมัติ, ทบทวน SOP
  - บันทึกการอ่าน/อบรม (Training Log) ของบุคลากร
- **บันทึก QC:** ฟอร์มบันทึก QC ประจำวัน
- **เอกสารคุณภาพ (QC Documents):**
  - อัปโหลด/จัดเก็บผล IQC, EQA, Inter-lab
  - [เพิ่มเติม] เอกสารทวนสอบ/วาลิเดต (Method Validation)
  - [เพิ่มเติม] เอกสาร Change Control

---

## 6. เมนู: รายงานและสถิติ (Reports & Analytics)

- **Dashboard สรุปผู้บริหาร:**
  - [เพิ่มเติม] รายงานติดตาม TAT (Real-time): แยก pre/analytic/post และรายงานอัตราการเกินกำหนด SLA
- **ระบบ Export สถิติ (รายเดือน/ปี):**
  - สถิติจำนวนเคส (แยกตามรายการตรวจ, หน่วยงาน)
  - **[เพิ่มเติม] รายงาน KPI คุณภาพ:**
    - อัตราการปฏิเสธสิ่งส่งตรวจ (Rejection Rate)
    - อัตราการตรวจซ้ำ (Repeat Rate)
    - อัตราการเกิด ADR (หากมีการติดตามผล)
  - **[เพิ่มเติม] รายงานการนำไปใช้ (Adoption Metrics):**
    - จำนวนหน่วยบริการที่เข้าร่วม
    - จำนวนบุคลากรที่ผ่านการอบรม

## 7. เมนู: การตั้งค่าระบบ (Admin Panel)

- **การจัดการผู้ใช้งาน (User Management):** (เพิ่ม/ลบ/แก้ไข ผู้ใช้งานและกำหนดสิทธิ์) ✔️
- **[เพิ่มเติม] การจัดการ PDPA (PDPA Management):**
  - ตั้งค่า e-Consent (dynamic, ถอน, ผู้แทนโดยชอบธรรม) ✔️
  - ตั้งค่าเทมเพลต Consent และวัตถุประสงค์ (Clinical/Research) ✔️
  - ตั้งค่านโยบาย (เช่น ระยะเวลาเก็บรักษา Retention Policy)
  - แบบประเมินความเสี่ยง (DPIA)
  - บันทึกร่องรอยการประมวลผล (Records of Processing) ✔️
  - กระบวนการแจ้งเหตุละเมิดข้อมูล
- **[เพิ่มเติม] การเชื่อมต่อระบบ (Integration Settings):**
  - ตั้งค่าการเชื่อมต่อ HIS/EMR (ADT/ORD/ORU/FHIR)
  - ตั้งค่าการเชื่อมต่อเครื่องมือ Analyzer (ASTM/HL7)
  - ตั้งค่า API (สำหรับแล็บภายนอก)
- **การตั้งค่าทั่วไป (System Settings):** (โลโก้, ข้อความ ISO)
- **[เพิ่มเติม] การจัดการสิทธิ์ (License & TT):**
  - หน้าข้อกำหนดสิทธิการใช้งาน (Licensing Terms) ✔️
  - โมดูลสนับสนุนการถ่ายทอดเทคโนโลยี (TT)
- **Log การใช้งานระบบ (Audit Log):** (บันทึกการเข้าถึง/แก้ไขข้อมูลทั้งหมด) ✔️
